# 插件消息皮肤 · 引用 UI 适配设计 Spec

## 正确范围

本轮设计对象不是 Octo Web 源码中的通用引用组件，而是 `octo-help` 浏览器插件在 `utils/octoBeautify.ts` 内注入的消息皮肤 CSS。目标站 `Mininglamp-OSS/octo-web` 只作为 DOM 与字段事实来源，生产阶段不向该仓库提交代码。需要适配的消息主题是插件弹窗“消息主题（美化换肤）”中的三项：赛博紫·亮、赛博紫·暗、美加墨世界杯。全站配色和足球射门动画不是本轮设计对象，但引用 UI 必须与对应消息气泡共存，不能破坏现有 AI、自己、他人三档身份色，也不能与世界杯气泡底部的足球动画区域争夺空间。

## 共同组件策略

三个皮肤使用同一引用信息架构：引用发送者名称、可选 Space 后缀、单行摘要、整块点击定位。结构继续兼容新版 `.wk-reply-block` 与 legacy `.wk-message-text-reply`。引用与当前消息必须组成一个视觉单元：外层消息组承担唯一边框、身份色轨道、圆角、阴影和宽度；引用区域作为组内头部，正文作为组内主体，两者之间仅保留一条发丝分隔线。共同布局采用紧凑两行引用头部：第一行 12px 来源名称，第二行 12px 摘要；摘要单行省略。引用关系通过组内左侧“引用括号”表达，它是一条带上下短横的规则线，可利用新版已有的 `__bar` 节点实现；legacy 版本用伪元素补齐。引用区域 hover 仅增强自身底色和括号，不移动整个消息组，不增加永久说明文字。

## 三套皮肤适配

### 赛博紫·亮

底色延续现有冷白卡与冷紫 gamer 气质。引用表面使用非常浅的冷灰紫，边界为发丝线，引用括号跟随当前消息身份色：AI 紫 `#7C6BF0`、自己绿 `#46A877`、他人灰蓝 `#9AA0C2`。来源名比摘要高一个字重，摘要使用 `#888C9C`。只保留一处小型 HUD 语言，即括号末端的短横，不再给引用块复制正文气泡的四角取景框，避免两套装饰争抢。

### 赛博紫·暗

结构与亮色完全一致，降低维护成本，但不能简单反色。引用表面采用比消息气泡稍亮的深紫灰，边界使用低透明白，AI/自己/他人的身份色提高亮度与饱和度以满足暗底识别。来源文字接近 `#D7D9E5`，摘要约 `#9CA0B3`。hover 只提升表面亮度并让括号更清晰，不使用大面积霓虹光晕。

### 美加墨世界杯

保留体育画报与主办国珠宝色体系，但移除当前引用块的完整球门框、菱形网格和 `octo-net-ripple` 抖动。引用表面为暖纸色，正文为暖墨，身份括号分别使用 AI 藏蓝 `#13294B`、自己深红 `#C8102E`、他人松绿 `#0B6E4F`。括号内侧增加一条非常短的金色 trim，引用底部保留 1px 金色规则线，与现有消息气泡的金色底边呼应。它表达“赛事皮肤”，但不把引用内容变成体育器材。

## 输出与验收

输出为三个独立 HTML 和一个比较入口，尺寸以 1440×900 桌面端为主。每版同时展示 AI、自己、他人三类带引用消息，并使用真实中文长摘要测试省略。需在 1024×768 下无横向溢出，引用点击有定位反馈，控制台错误为零。用户确认后再把规则映射回插件 CSS；当前阶段不修改 `utils/octoBeautify.ts`。
